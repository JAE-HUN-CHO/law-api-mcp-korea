# 학칙ㆍ공단ㆍ공공기관 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 학칙ㆍ공단ㆍ공공기관 본문 조회
- **기본 설명**: 학칙(school), 공단(public), 공공기관(pi) 규정 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=schlPubRulInfoGuide

## 3. 요청 URL (Endpoint)
### 3-1. 학칙 본문 조회
- http://www.law.go.kr/DRF/lawService.do?target=school

### 3-2. 공단 본문 조회
- http://www.law.go.kr/DRF/lawService.do?target=public

### 3-3. 공공기관 본문 조회
- http://www.law.go.kr/DRF/lawService.do?target=pi

| target | string : school(필수) | school/public/pi 중 하나 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=school&OC=test&type=XML&ID={법령ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=school&OC=test&type=JSON&ID={법령ID}
```
