# 일상용어 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 일상용어 조회
- **기본 설명**: 법령정보지식베이스의 일상용어 조회 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=dlytrmGuide

## 3. 요청 URL (Endpoint)
- https://www.law.go.kr/DRF/lawSearch.do?target=dlytrm

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : dlytrm(필수) | 서비스 대상 |
| type | char, 필수 | 출력 형태 : XML/JSON |
| query | string | 검색 질의 또는 대상 식별값 |
| display | int | 검색된 결과 개수(default=20, max=100) |
| page | int | 검색 결과 페이지(default=1) |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| target | string | 검색서비스 대상 |
| 키워드 | string | 검색 질의 |
| 검색결과개수 | int | 검색 건수 |
| page | int | 현재 페이지번호 |
| 일상용어 id | string | 일상용어 순번 |
| 일상용어명 | string | 일상용어명 |
| 설명 | string | 설명 |
| 관련법령용어링크 | string | 법령용어 연계 상세링크 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.

#### 요청 예시
- 예시 1 (XML)
```text
https://www.law.go.kr/DRF/lawSearch.do?target=dlytrm&OC=test&type=XML&query=개인정보&display=20&page=1
```
- 예시 2 (JSON)
```text
https://www.law.go.kr/DRF/lawSearch.do?target=dlytrm&OC=test&type=JSON&query=개인정보&display=20&page=1
```
#### 응답 예시
- XML 구조 예시
