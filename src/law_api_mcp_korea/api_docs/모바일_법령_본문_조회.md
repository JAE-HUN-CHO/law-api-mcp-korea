# 모바일 법령 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 모바일 법령 본문 조회
- **기본 설명**: 모바일 전용 형식의 법령 본문 조회 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=mobLsInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=law&mobileYn=Y

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : law, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML |
| mobileYn | char : Y, 필수 | 모바일 여부 |
| ID | char | 법령 ID (ID 또는 MST 중 하나 필수) |
| MST | char | 법령 마스터 번호 |
| LM | string | 법령명 |
| LD | int | 공포일자 |
| LN | int | 공포번호 |
| JO | int:6 | 조번호 |
| PD | char | 부칙 표시 |
| PN | int | 부칙번호 |
| BD | char | 별표 표시 |
| BT | int | 별표구분 |
| BN | int | 별표번호 |
| BG | int | 별표가지번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 기본정보 | string | 기본 메타데이터 영역 |
| 법령ID | int | 법령 ID |
| 법령일련번호 | int | 법령일련번호 |
| 법령명 | string | 법령명 |
| 법종구분 | string | 법종구분 |
| 공포일자 | int | 공포일자 |
| 공포번호 | int | 공포번호 |
| 시행일자 | int | 시행일자 |
| 제개정구분 | string | 제개정구분 |
| 조문내용 | string | 조문/본문 |
| 부칙 | string | 부칙 |
| 별표 | string | 별표/서식 |
| 상세링크/첨부 | string | 관련 링크 및 첨부 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (HTML)
```text
http://www.law.go.kr/DRF/lawService.do?target=law&mobileYn=Y&OC=test&type=HTML&ID={법령ID}
```
#### 응답 예시
- HTML 구조 예시
```html
<!DOCTYPE html>
<html>
  <body>
    <h1>모바일 법령 본문 조회</h1>
    <div data-field="기본정보">예시값</div>
    <div data-field="법령ID">12345</div>
    <div data-field="법령일련번호">12345</div>
    <div data-field="법령명">자동차관리법</div>
  </body>
</html>
```

## 8. 메타
- 정리 기준: 국가법령정보 공동활용 OPEN API 활용가이드 구조화 요약
- 근거 수준: direct
