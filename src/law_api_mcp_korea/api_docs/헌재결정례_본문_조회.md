# 헌재결정례 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 헌재결정례 본문 조회
- **기본 설명**: 헌재결정례 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=detcInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=detc

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : detc, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char, 필수 | 대상 일련번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 헌재결정례일련번호 | int | 헌재결정례일련번호 |
| 사건명 | string | 사건명 |
| 사건번호 | string | 사건번호 |
| 선고일자 | string | 선고일자 |
| 결정요지 | string | 결정요지 |
| 주문 | string | 주문 |
| 이유 | string | 이유 |
| 본문 | string | 본문 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=detc&OC=test&type=XML&ID={헌재결정례일련번호}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=detc&OC=test&type=JSON&ID={헌재결정례일련번호}
```
#### 응답 예시
- XML 구조 예시
