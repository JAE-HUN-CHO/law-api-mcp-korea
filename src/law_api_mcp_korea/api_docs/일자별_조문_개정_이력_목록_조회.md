# 일자별 조문 개정 이력 목록 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 일자별 조문 개정 이력 목록 조회
- **기본 설명**: 지정 일자 또는 기간의 조문 개정 이력을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=lsDayJoRvsListGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawSearch.do?target=lsJoHstInf

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string(필수) | 신청한 API인증값 |
| target | string : lsJoHstInf(필수) | 서비스 대상 |
| type | char(필수) | 출력 형태 XML/JSON |
| regDt | int | 조문 개정일, 8자리 (20150101) |
| fromRegDt | int | 조회기간 시작일, 8자리 (20150101) |
| toRegDt | int | 조회기간 종료일, 8자리 (20150101) |
| ID | int | 법령ID |
| JO | int | 조문번호 조문번호 4자리 + 조 가지번호 2자리 (000202 : 제2조의2) |
| org | string | 소관부처별 검색(소관부처코드 제공) |
| page | int | 검색 결과 페이지 (default=1) |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://law.go.kr/DRF/lawSearch.do?target=lsJoHstInf&OC=test&regDt=20250401&type=XML
```
- 예시 2 (XML)
```text
http://www.law.go.kr/DRF/lawSearch.do?target=lsJoHstInf&OC=test&fromRegDt=20250101&toRegDt=20250131&type=XML
```
- 예시 3 (JSON)
```text
http://www.law.go.kr/DRF/lawSearch.do?target=lsJoHstInf&OC=test&regDt=20250401&org=1352000&type=JSON
```

#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="target">예시값</field>
    <field name="regDt/fromRegDt/toRegDt">1</field>

    <field name="조문번호">12345</field>
    <field name="상세링크">https://www.law.go.kr/...</field>
    <field name="개정일자">20250501</field>
  </item>
</response>
```
- JSON 구조 예시
```json

  "data": [

      "target": "예시값",

      "상세링크": "https://www.law.go.kr/...",

```
