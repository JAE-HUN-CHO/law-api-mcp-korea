# 현행법령(공포일) 본문 조항호목 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 현행법령(공포일) 본문 조항호목 조회
- **기본 설명**: 현행법령(공포일 기준)의 조·항·호·목 단위 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=lsNwJoListGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=lawjosub

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : lawjosub, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char | 법령 ID (ID 또는 MST 중 하나 필수) |
| MST | char | 법령 마스터 번호(lsi_seq) |
| efYd | int | 시행일자(MST 사용 시 필수) |
| JO | char, 필수 | 조번호 6자리 |
| HANG | char | 항번호 |
| HO | char | 호번호 |
| MOK | char | 목번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 기본정보 | string | 법령 기본정보 |
| 법령ID | int | 법령ID |
| 법령명 | string | 법령명 |
| 조번호 | int | 조번호 |
| 조가지번호 | int | 조가지번호 |
| 조제목 | string | 조제목 |
| 조내용 | string | 조내용 |
| 항내용 | string | 항내용 |
| 호내용 | string | 호내용 |
| 목내용 | string | 목내용 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=lawjosub&OC=test&type=XML&ID={법령ID}&efYd=20151007&JO=000300
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=lawjosub&OC=test&type=JSON&ID={법령ID}&efYd=20151007&JO=000300
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="기본정보">예시값</field>
    <field name="법령ID">12345</field>
    <field name="법령명">자동차관리법</field>
    <field name="조번호">12345</field>
    <field name="조제목">자동차관리법</field>
    <field name="조내용">예시 본문</field>
  </item>
</response>
```
- JSON 구조 예시
```json
{
  "data": [
    {
      "기본정보": "예시값",
      "법령ID": 12345,
      "법령명": "자동차관리법",
      "조번호": 12345,
      "조제목": "자동차관리법",
      "조내용": "예시 본문"
    }
  ]
}
```

## 8. 메타

- 근거 수준: direct
