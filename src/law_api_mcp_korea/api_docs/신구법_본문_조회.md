# 신구법 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 신구법 본문 조회
- **기본 설명**: 신구법 비교 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=oldAndNewInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=oldAndNew

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : oldAndNew, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char | 신구법일련번호 또는 법령 ID |
| MST | char | 법령 마스터 번호 |
| LM | string | 법령명 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 기본정보 | string | 기본정보 |
| 현행조문 | string | 현행 조문 |
| 종전조문 | string | 종전 조문 |
| 비교영역 | string | 신구 대비 영역 |
| 상세링크 | string | 상세링크 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=oldAndNew&OC=test&type=XML&ID={법령ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=oldAndNew&OC=test&type=JSON&ID={법령ID}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="기본정보">예시값</field>
    <field name="현행조문">예시값</field>
    <field name="종전조문">예시값</field>
    <field name="비교영역">예시값</field>
    <field name="상세링크">https://www.law.go.kr/...</field>
  </item>
</response>
```
- JSON 구조 예시
```json
{
  "data": [
    {
      "기본정보": "예시값",
      "현행조문": "예시값",
      "종전조문": "예시값",
      "비교영역": "예시값",
      "상세링크": "https://www.law.go.kr/..."
    }
  ]
}
```

## 8. 메타

- 근거 수준: direct
