# 법령 체계도 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 법령 체계도 본문 조회
- **기본 설명**: 법령 체계도 상세를 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=lsStmdInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=lsStmd

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string(필수) | 신청한 API인증값 |
| target | string : lsStmd(필수) | 서비스 대상 |
| type | char(필수) | 출력 형태 : HTML/XML/JSON |
| ID | char | 법령 ID (ID 또는 MST 중 하나는 반드시 입력) |
| MST | char | 법령 마스터 번호 - 법령테이블의 lsi_seq 값을 의미함 |
| LM | string | 법령의 법령명(법령명 입력시 해당 법령 링크) |
| LD | int | 법령의 공포일자 |
| LN | int | 법령의 공포번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 기본정보 | string | 기본정보 |
| 법령ID | int | 법령ID |
| 법령일련번호 | int | 법령일련번호 |
| 공포일자 | int | 공포일자 |
| 공포번호 | int | 공포번호 |
| 법종구분 | string | 법종구분 |
| 법령명 | string | 법령 |
| 시행일자 | int | 시행일자 |
| 제개정구분 | string | 제개정구분 |
| 상하위법 | string | 상하위법 |
| 법률 | string | 법률 |
| 시행령 | string | 시행령 |
| 시행규칙 | string | 시행규칙 |
| 본문 상세링크 | string | 본문 상세링크 |
| 관련법령 | string | 관련법령 |
| 법종구분코드 | string | 법종구분코드 |
| 제개정구분코드 | string | 제개정구분코드 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (HTML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lsStmd&MST=142362&type=HTML
```
- 예시 2 (HTML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lsStmd&MST=142591&type=HTML
```
- 예시 3 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lsStmd&MST=142362&type=XML
```
- 예시 4 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lsStmd&MST=142362&type=JSON
```

#### 응답 예시
- XML 구조 예시
