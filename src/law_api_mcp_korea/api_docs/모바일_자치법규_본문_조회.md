# 모바일 자치법규 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 모바일 자치법규 본문 조회
- **기본 설명**: 모바일 전용 형식의 자치법규 본문 조회 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=mobOrdinInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=ordin

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string(필수) | 신청한 API인증값 |
| target | string : ordin(필수) | 서비스 대상 |
| ID | char | 자치법규 ID |
| MST | string | 자치법규 마스터 번호 |
| type | char | 출력 형태 : HTML |
| mobileYn | char : Y (필수) | 모바일여부 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 응답 본문 | HTML | 공식 가이드는 개별 응답 필드 명세표를 제공하지 않음 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (HTML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=ordin&ID=2047729&type=HTML&mobileYn=Y
```
- 예시 2 (HTML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=ordin&type=HTML&mobileYn=Y&MST=1062134
```

#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="자치법규일련번호">12345</field>
    <field name="자치법규명">서울특별시 주차장 조례</field>
    <field name="자치단체명">예시값</field>
    <field name="공포일자">20250501</field>
    <field name="조문내용">예시 본문</field>
    <field name="공포번호">12345</field>
  </item>
</response>
```
- JSON 구조 예시
```json
{
  "data": [
    {
      "자치법규일련번호": 12345,
      "자치법규명": "서울특별시 주차장 조례",
      "자치단체명": "예시값",
      "공포일자": 20250501,
      "조문내용": "예시 본문",
      "공포번호": "12345"
    }
  ]
}
```

## 8. 메타
- 정리 기준: 국가법령정보 공동활용 OPEN API 활용가이드 구조화 요약
- 근거 수준: direct
