# 감사원 사전컨설팅 의견서 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 감사원 사전컨설팅 의견서 본문 조회
- **기본 설명**: 감사원 사전컨설팅 의견서 본문 조회 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=baiPvcsInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=baiPvcs

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : baiPvcs, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char, 필수 | 감사원사전컨설팅의견서일련번호 |
| LM | string | 의견서명 |
| fields | string | 응답항목 옵션 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 감사원사전컨설팅의견서일련번호 | int | 의견서 일련번호 |
| 의견서명 | string | 의견서명 |
| 회신일자 | string | 회신일자 |
| 요청기관명 | string | 요청기관명 |
| 접수번호 | string | 접수번호 |
| 개요 | string | 개요 |
| 요지 | string | 요지 |
| 판단기준 | string | 판단기준 |
| 검토결과 | string | 검토결과 |
| 종합의견 | string | 종합의견 |
| 데이터기준일자 | string | 데이터기준일자 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=baiPvcs&OC=test&type=XML&ID={의견서일련번호}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=baiPvcs&OC=test&type=JSON&ID={의견서일련번호}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="감사원사전컨설팅의견서일련번호">12345</field>

    <field name="회신일자">20250501</field>
    <field name="요청기관명">예시 기관</field>
    <field name="접수번호">12345</field>

```
