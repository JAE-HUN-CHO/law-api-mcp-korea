# 맞춤형 자치법규 조문 목록 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 맞춤형 자치법규 조문 목록 조회
- **기본 설명**: 맞춤형 분류코드(vcode)에 따른 자치법규 조문 목록을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=custOrdinJoListGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawSearch.do?target=couseOrdin

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string(필수) | 신청한 API인증값 |
| target | string : couseOrdin(필수) | 서비스 대상 |
| type | char(필수) | 출력 형태 : HTML/XML/JSON |
| vcode | string | 분류코드(필수) 자치법규는 O로 시작하는 14자리 코드(O0000000000001) |
| lj | string(필수) | 조문여부 (jo 고정) |
| display | int | 검색된 결과 개수 (default=20 max=100) |
| page | int | 검색 결과 페이지 (default=1) |
| popYn | string | 상세화면 팝업창 여부(팝업창으로 띄우고 싶을 때만 'popYn=Y') |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 7. 비고
- lj=jo 옵션을 추가하여 조문 목록까지 확장 조회합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawSearch.do?OC=test&target=couseOrdin&type=XML&lj=jo&vcode=O0000000000602
```
- 예시 2 (HTML)
```text
http://www.law.go.kr/DRF/lawSearch.do?OC=test&target=couseOrdin&type=HTML&lj=jo&vcode=O0000000000602
```
- 예시 3 (JSON)
```text
http://www.law.go.kr/DRF/lawSearch.do?OC=test&target=couseOrdin&type=JSON&lj=jo&vcode=O0000000000602
```

#### 응답 예시
