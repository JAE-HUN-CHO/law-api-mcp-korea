# 삭제 데이터 목록 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 삭제 데이터 목록 조회
- **기본 설명**: 삭제 데이터 목록을 조회하는 API. 가이드 본문에는 target 값 설명이 datDel로 병기됨

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=datDelHstGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawSearch.do?target=delHst

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : datDel / 요청 URL 표기상 delHst | 서비스 대상 |
| type | char, 필수 | 출력 형태 : XML/JSON |
| display | int | 검색 개수 |
| page | int | 결과 페이지 |
| knd | string | 삭제 데이터 구분 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| target | string | 검색 대상 |
| 삭제구분 | string | 삭제 데이터 구분 |
| 일련번호/ID | string | 삭제 대상 식별자 |
| 제목/명칭 | string | 삭제 대상 제목 |
| 삭제일자 | int | 삭제일자 |
| 데이터기준일시 | string | 데이터기준일시 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 7. 비고
- 가이드 본문에서 요청 URL 표기와 target 설명(delHst / datDel)이 혼재되어 있습니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawSearch.do?target=delHst&OC=test&type=XML&display=20&page=1&knd={구분코드}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawSearch.do?target=delHst&OC=test&type=JSON&display=20&page=1&knd={구분코드}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="target">예시값</field>
    <field name="삭제구분">예시값</field>
    <field name="일련번호/ID">12345</field>
    <field name="제목/명칭">자동차관리법</field>
    <field name="삭제일자">20250501</field>
    <field name="데이터기준일시">2025-05-01 09:00:00</field>
  </item>
</response>
```
- JSON 구조 예시
```json
{
  "data": [
    {
      "target": "예시값",
      "삭제구분": "예시값",
      "일련번호/ID": "12345",
      "제목/명칭": "자동차관리법",
      "삭제일자": 20250501,

```
