# 중앙환경분쟁조정위원회 결정문 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 중앙환경분쟁조정위원회 결정문 본문 조회
- **기본 설명**: 중앙환경분쟁조정위원회 결정문 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=eccInfoGuide

## 3. 요청 URL (Endpoint)
- https://www.law.go.kr/DRF/lawService.do?target=ecc

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string(필수) (중앙환경분쟁조정위원회 : ecc) | 서비스 대상(위원회별 target 코드) |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char, 필수 | 결정문 일련번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 결정문일련번호 | int | 결정문 일련번호 |
| 기관명 | string | 기관명 |
| 사건번호 | string | 사건번호 |
| 자료구분 | string | 자료구분 |
| 담당부서 | string | 담당부서 |
| 등록일 | string | 등록일 |
| 제목 | string | 제목 |
| 내용 | string | 내용 |
| 판정사항 | string | 판정사항 |
| 판정요지 | string | 판정요지 |
| 판정결과 | string | 판정결과 |
| 각주번호 | int | 각주번호 |
| 각주내용 | string | 각주내용 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 7. 비고
- 노동위원회(nlrc) 대표 가이드 구조를 기준으로 동일 계열 위원회 API를 정규화했습니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
https://www.law.go.kr/DRF/lawService.do?target=ecc&OC=test&type=XML&ID={결정문일련번호}
```
- 예시 2 (JSON)
```text
https://www.law.go.kr/DRF/lawService.do?target=ecc&OC=test&type=JSON&ID={결정문일련번호}
```
#### 응답 예시
- XML 구조 예시
