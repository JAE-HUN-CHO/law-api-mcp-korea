# 위원회 결정문 본문 조회 (공정거래위원회·국민권익위원회·개인정보보호위원회)

## 1. API 명칭 및 기본 설명
- **API 명칭**: 위원회 결정문 본문 조회 (공정거래위원회·국민권익위원회·개인정보보호위원회)
- **기본 설명**: 공정거래위원회(ftc), 국민권익위원회(acr), 개인정보보호위원회(ppc)의 결정문 본문 조회를 한 페이지에 묶어 제공하는 가이드

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=cmtInfoGuide

## 3. 요청 URL (Endpoint)
### 3-1. 공정거래위원회(ftc) 결정문 본문
- https://www.law.go.kr/DRF/lawService.do?target=ftc

### 3-2. 국민권익위원회(acr) 결정문 본문
- https://www.law.go.kr/DRF/lawService.do?target=acr

### 3-3. 개인정보보호위원회(ppc) 결정문 본문
- https://www.law.go.kr/DRF/lawService.do?target=ppc

| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : ftc(필수) | 서비스 대상(ftc/acr/ppc) |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char, 필수 | 결정문 일련번호 |

| ID | char, 필수 | 결정문 일련번호 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 7. 비고
- 공정거래위원회에는 시정권고서 영역이 함께 안내됩니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

### 공정거래위원회(ftc) 결정문 본문
#### 요청 예시
- 예시 1 (XML)
```text
https://www.law.go.kr/DRF/lawService.do?target=ftc&OC=test&type=XML&ID={결정문일련번호}
```
- 예시 2 (HTML)
```text
https://www.law.go.kr/DRF/lawService.do?target=ftc&OC=test&type=HTML&ID={결정문일련번호}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="결정문일련번호">12345</field>
    <field name="기관명">예시 기관</field>
    <field name="제목">예시 사건</field>
    <field name="사건번호">12345</field>
    <field name="본문">예시 본문</field>
    <field name="이유">예시 본문</field>
  </item>
</response>
```
