# 일자별 조문 개정 이력 목록 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 일자별 조문 개정 이력 목록 조회
- **기본 설명**: 지정 일자 또는 기간의 조문 개정 이력을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=lsDayJoRvsListGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawSearch.do?target=lsJoHstInf

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : lsJoHstInf, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : XML/JSON |
| regDt | int | 기준일자 |
| fromRegDt | int | 시작일자 |
| toRegDt | int | 종료일자 |
| ID | char | 법령 ID |
| JO | char | 조문번호 |
| org | string | 소관부처 코드 |
| page | int | 검색 결과 페이지 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| target | string | 검색서비스 대상 |
| regDt/fromRegDt/toRegDt | int | 조회 기준일 |
| 법령ID | int | 법령ID |
| 조문번호 | string | 조문번호 |
| 개정일자 | int | 개정일자 |
| 개정유형 | string | 개정유형 |
| 상세링크 | string | 상세 링크 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawSearch.do?target=lsJoHstInf&OC=test&type=XML&regDt=20250501&page=1
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawSearch.do?target=lsJoHstInf&OC=test&type=JSON&regDt=20250501&page=1
```
