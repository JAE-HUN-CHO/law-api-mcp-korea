# 법령 체계도 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 법령 체계도 본문 조회
- **기본 설명**: 법령 체계도 상세를 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=lsStmdInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=lsStmd

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : lsStmd, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char | 법령 ID |
| MST | char | 법령 마스터 번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 기본정보 | string | 법령 기본정보 |
| 체계도 | string | 상위·하위 체계 구조 |
| 상위법령목록 | string | 상위 법령 목록 |
| 하위법령목록 | string | 하위 법령 목록 |
| 상세링크 | string | 체계도 상세링크 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=lsStmd&OC=test&type=XML&ID={법령ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=lsStmd&OC=test&type=JSON&ID={법령ID}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="기본정보">예시값</field>
    <field name="체계도">예시값</field>
    <field name="상위법령목록">예시값</field>
    <field name="하위법령목록">예시값</field>
    <field name="상세링크">https://www.law.go.kr/...</field>
  </item>
</response>
```
- JSON 구조 예시
```json
{
  "data": [
    {
      "기본정보": "예시값",
      "체계도": "예시값",
      "상위법령목록": "예시값",
      "하위법령목록": "예시값",
      "상세링크": "https://www.law.go.kr/..."
    }
  ]
}
```
