# 법령 용어 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 법령 용어 본문 조회
- **기본 설명**: 법령 용어 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=lsTrmInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=lstrm

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string(필수) | 신청한 API인증값 |
| target | string : lstrm(필수) | 서비스 대상 |
| type | char(필수) | 출력 형태 : HTML/XML/JSON |
| query | string | 상세조회하고자 하는 법령용어 명 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 법령용어 일련번호 | int | 법령용어 일련번호 |
| 법령용어명_한글 | string | 법령용어명 한글 |
| 법령용어명_한자 | string | 법령용어명한자 |
| 법령용어코드 | int | 법령용어코드 |
| 법령용어코드명 | string | 법령용어코드명 |
| 출처 | string | 출처 |
| 법령용어정의 | string | 법령용어정의 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (HTML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lstrm&query=선박&type=HTML
```
- 예시 2 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lstrm&query=선박&type=XML
```
- 예시 3 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?OC=test&target=lstrm&query=선박&type=JSON
```

#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="법령용어일련번호">12345</field>
    <field name="법령용어명">예시값</field>
    <field name="정의">예시값</field>
    <field name="비고">예시 본문</field>
    <field name="관련법령">예시값</field>
    <field name="관련조문">예시값</field>
  </item>
</response>
```
- JSON 구조 예시
```json

      "법령용어명": "예시값",
```
