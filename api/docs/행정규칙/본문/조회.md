# 행정규칙 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 행정규칙 본문 조회
- **기본 설명**: 행정규칙 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=admrulInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=admrul

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : admrul, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char | 행정규칙 일련번호 |
| LID | char | 행정규칙 ID |
| LM | string | 조회하고자 하는 정확한 행정규칙명 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=admrul&OC=test&type=XML&ID={행정규칙ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=admrul&OC=test&type=JSON&ID={행정규칙ID}
```
#### 응답 예시
