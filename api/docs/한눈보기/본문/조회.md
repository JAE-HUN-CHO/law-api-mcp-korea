# 한눈보기 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 한눈보기 본문 조회
- **기본 설명**: 한눈보기 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=oneViewInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=oneview

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : oneview, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char, 필수 | 한눈보기 일련번호 또는 법령 ID |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| oneview일련번호 | int | 한눈보기 일련번호 |
| 법령명 | string | 법령명 |
| 요약정보 | string | 한눈보기 요약 |
| 표/도식 | string | 시각화 정보 |
| 본문 | string | 한눈보기 본문 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=oneview&OC=test&type=XML&ID={법령ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=oneview&OC=test&type=JSON&ID={법령ID}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="oneview일련번호">12345</field>
    <field name="법령명">자동차관리법</field>
    <field name="요약정보">예시값</field>
    <field name="표/도식">예시값</field>
    <field name="본문">예시 본문</field>
  </item>
</response>
```
- JSON 구조 예시
```json
{
  "data": [
    {
      "oneview일련번호": 12345,
      "법령명": "자동차관리법",
      "요약정보": "예시값",
      "표/도식": "예시값",
      "본문": "예시 본문"
    }
  ]
}
```

## 8. 메타
- 정리 기준: 국가법령정보 공동활용 OPEN API 활용가이드 구조화 요약
- 근거 수준: direct
