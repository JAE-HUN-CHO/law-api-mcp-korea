# 위임법령 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 위임법령 조회
- **기본 설명**: 가이드 목록에는 위임법령 조회로 노출되며, 실제 요청/응답 구조는 3단 비교 본문 API의 위임조문(knd=2) 모드와 동일하게 제공됨

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=thdCmpInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=thdCmp

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : thdCmp, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| knd | int, 필수 | 3단 비교 종류(1=인용조문, 2=위임조문) |
| ID | char | 법령 ID (ID 또는 MST 중 하나 필수) |
| MST | char | 법령 마스터 번호 |
| LM | string | 법령명 |
| LD | int | 공포일자 |
| LN | int | 공포번호 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 기본정보 | string | 위임 삼단비교 기본정보 |
| 법령ID | int | 법령 ID |
| 법령일련번호 | int | 법령일련번호 |
| 공포일자 | int | 공포일자 |
| 공포번호 | int | 공포번호 |
| 법종구분 | string | 법종 구분 |
| 법령명 | string | 법령 명 |
| 시행일자 | int | 시행일자 |
| 제개정구분 | string | 제개정구분 |
| 삼단비교존재여부 | int | 삼단비교 존재 여부 |
| 기준법령목록 | string | 기준법 법령 목록 |
| 위임3비교상세링크 | string | 위임조문 3비교 상세링크 |
| 법률조문 | string | 법률조문 |
| 조번호 | int | 조번호 |

| 조내용 | string | 조내용 |
| 시행령조문 | string | 하위 시행령조문 |
| 시행규칙조문목록 | string | 시행규칙 조문목록 |
| 시행규칙조문 | string | 하위 시행규칙조문 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 7. 비고
- 가이드 목록에는 별도 항목으로 노출되지만, 실제 요청/응답 구조는 3단 비교 본문 조회의 위임조문(knd=2) 모드와 동일한 것으로 정리했습니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=thdCmp&OC=test&type=XML&knd={구분코드}&ID={법령ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=thdCmp&OC=test&type=JSON&knd={구분코드}&ID={법령ID}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="기본정보">예시값</field>
    <field name="법령ID">12345</field>
    <field name="법령일련번호">12345</field>
    <field name="공포일자">20250501</field>
    <field name="법령명">자동차관리법</field>
    <field name="조제목">자동차관리법</field>
  </item>
</response>
```
- JSON 구조 예시
```json

  "data": [
    {
      "기본정보": "예시값",
      "법령ID": 12345,
      "법령일련번호": 12345,
      "공포일자": 20250501,
      "법령명": "자동차관리법",

    }

```
