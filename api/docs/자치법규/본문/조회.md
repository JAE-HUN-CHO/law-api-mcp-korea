# 자치법규 본문 조회

## 1. API 명칭 및 기본 설명
- **API 명칭**: 자치법규 본문 조회
- **기본 설명**: 자치법규 본문을 조회하는 API

## 2. 가이드 페이지 URL
- https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=ordinInfoGuide

## 3. 요청 URL (Endpoint)
- http://www.law.go.kr/DRF/lawService.do?target=ordin

## 4. 요청 변수 (Request Parameters) 명세표
| 변수명 | 타입/필수 여부 | 설명 |
| --- | --- | --- |
| OC | string, 필수 | 사용자 이메일 ID |
| target | string : ordin, 필수 | 서비스 대상 |
| type | char, 필수 | 출력 형태 : HTML/XML/JSON |
| ID | char | 자치법규 일련번호 |
| LID | char | 자치법규 ID |
| LM | string | 정확한 자치법규명 |

## 5. 출력 결과 (Response Elements) 명세표
| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| 자치법규일련번호 | int | 자치법규일련번호 |
| 자치법규명 | string | 자치법규명 |
| 자치단체명 | string | 자치단체명 |
| 공포일자 | int | 공포일자 |
| 공포번호 | string | 공포번호 |
| 제개정구분명 | string | 제개정구분명 |
| 자치법규ID | int | 자치법규 ID |
| 조문내용 | string | 조문내용 |
| 부칙 | string | 부칙 |
| 별표 | string | 별표 |
| 첨부파일 | string | 첨부파일 |

## 6. 에러 코드
- 가이드 페이지에 별도 에러 코드 표는 명시되지 않은 경우가 많습니다. 일반적으로 HTTP 오류, 빈 결과, 또는 XML/JSON 응답의 상태값(제공 시)으로 실패를 판별합니다.

## 샘플 요청 및 응답 예시
- 아래 예시는 호출 형태를 빠르게 확인하기 위한 템플릿입니다.
- `{...}` 형태의 식별자 값은 실제 조회 대상 값으로 치환해서 사용합니다.

#### 요청 예시
- 예시 1 (XML)
```text
http://www.law.go.kr/DRF/lawService.do?target=ordin&OC=test&type=XML&ID={자치법규ID}
```
- 예시 2 (JSON)
```text
http://www.law.go.kr/DRF/lawService.do?target=ordin&OC=test&type=JSON&ID={자치법규ID}
```
#### 응답 예시
- XML 구조 예시
```xml
<response>
  <item>
    <field name="자치법규일련번호">12345</field>
    <field name="자치법규명">서울특별시 주차장 조례</field>
    <field name="자치단체명">예시값</field>
    <field name="공포일자">20250501</field>
    <field name="조문내용">예시 본문</field>
    <field name="공포번호">12345</field>

```
- JSON 구조 예시
```json

      "자치단체명": "예시값",

```

## 8. 메타
